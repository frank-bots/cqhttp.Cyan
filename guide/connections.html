    连接方式

    常用连接方式

    cqhttp.Cyan支持通过全部三种连接方式连接cqhttp插件，三种连接方式分别对应继承于CQApiClient的三个子类：

    类名 构造函数参数 连接方式
    CQHTTPClient string access_url,
    string access_token,
    int listen_port,
    string secret
    HTTP调用API，post_url消息上报
    CQWebsocketClient string access_url,
    string access_token,
    string event_url
    cqhttp.Cyan作为websocket客户端
    连接cqhttp调用API，接收上报消息
    CQReverseWSClient int bind_port,
    string api_path,
    string event_path,
    string access_token
    cqhttp.Cyan作为websocket服务端
    接受来自cqhttp的连接调用API，接收上报消息

    此外，CQApiClient都有use_group_table与use_message_table两参数，后文会说明其用途。

    进行组合

    如果需要进行"使用HTTP方式进行API调用，但使用Websocket监听上报事件"这样奇怪的操作的话，可以直接实例化CQApiClient类。CQApiClient的构造参数接收一个ICaller与一个IListener，分别用于调用API接口与监听上报事件。

    协议端的使用

    目前（May 2021）维护中的协议端有 onebot-kotlin，go-cqhttp 等。关于协议端的使用请参考其各自的README及文档。如果想要一键启动，可以：

    git clone https://github.com/frank-bots/cqhttp.Cyan.Demo
cd cqhttp.Cyan.Demo
# 修改配置文件
docker-compose up -d
